# ScopeFlow Thai - Beta Test Checklist

ใช้เช็กลิสต์นี้เพื่อทดสอบฟังก์ชันทั้งหมดก่อนการเปิดให้ผู้ใช้จริงทดสอบ (Beta Trial)

## 1. การติดตั้งและการเริ่มต้น (Setup & Startup)
- [ ] ติดตั้งและเปิดแอปพลิเคชัน ScopeFlow Thai สำเร็จ
- [ ] ที่หน้า Welcome Screen เลือก "สร้าง Demo Workspace" ไปที่โฟลเดอร์ว่างที่เตรียมไว้
- [ ] ตรวจสอบว่าระบบสร้าง Demo Workspace สำเร็จ และเปิดหน้า Overview ขึ้นมา

## 2. การจัดการโปรเจกต์เบื้องต้น (Project Management)
- [ ] ไปที่เมนู Health Check รันการตรวจสอบและพบว่าสถานะผ่านทั้งหมด
- [ ] สร้างลูกค้าใหม่ (New Client) สำเร็จ (ใส่ชื่อและผู้ติดต่อ)
- [ ] สร้างโปรเจกต์ใหม่ (New Project) ภายใต้ลูกค้าที่เพิ่งสร้างสำเร็จ
- [ ] เข้าไปที่โปรเจกต์ใหม่และเปิดดูหน้า Project Overview

## 3. การสร้างและแก้ไขเอกสาร (Document Editing)
- [ ] ไปที่แท็บ Quotation
- [ ] สร้าง Quotation ใหม่โดยใช้แบบฟอร์มที่มีให้
- [ ] ทดสอบกรอกข้อมูล ทดสอบการคำนวณ VAT และส่วนลด ว่าแสดงผลถูกต้อง
- [ ] ไปที่แท็บ Change Requests (CR)
- [ ] สร้าง CR ใหม่ พร้อมกรอกรายละเอียดการเปลี่ยนแปลง

## 4. ระบบการอนุมัติ (Approvals & Locks)
- [ ] กลับไปที่เอกสารที่เพิ่งสร้าง (เช่น Quotation)
- [ ] เพิ่มบันทึกการอนุมัติ (Approval Record) และแนบไฟล์หลักฐาน (Evidence)
- [ ] ยืนยันการล็อกเอกสาร (Lock Document) หลังจากเพิ่มการอนุมัติ
- [ ] ตรวจสอบว่าเอกสารที่ถูกล็อก ไม่สามารถแก้ไขข้อมูลได้แล้ว
- [ ] ลองกดสร้างการแก้ไขใหม่ (Create Revision) จากเอกสารที่ล็อกแล้ว ตรวจสอบว่าระบบสร้างเอกสารเวอร์ชันใหม่ให้แก้ไขได้

## 5. การสร้างชุดเอกสารและส่งออก (Export & Offline View)
- [ ] ไปที่เมนู "สร้าง Approval Pack" (Export Approval Pack)
- [ ] เลือกเอกสารที่ต้องการ และคลิกส่งออกไปเป็นไฟล์ HTML ภายใน Workspace
- [ ] เปิดไฟล์ HTML ที่ส่งออก ด้วยเว็บเบราว์เซอร์ปกติ แบบออฟไลน์
- [ ] ใช้ฟังก์ชัน Print ของระบบปฏิบัติการ (Ctrl+P หรือ Cmd+P) เลือก Save as PDF และตรวจสอบว่าเอกสารแสดงผลถูกต้องสวยงาม

## 6. การสำรองและกู้คืนข้อมูล (Backup & Restore)
- [ ] ไปที่ Settings -> Backup Workspace
- [ ] บันทึกไฟล์ ZIP สำรองไว้ที่โฟลเดอร์อื่น (นอก Workspace)
- [ ] ปิด Workspace ปัจจุบัน และไปที่หน้า Welcome Screen
- [ ] เลือก "เปิดจาก Backup" (Restore) เลือกไฟล์ ZIP ที่เพิ่งสำรอง และเลือกโฟลเดอร์ว่างโฟลเดอร์ใหม่
- [ ] เมื่อเปิดขึ้นมา ให้ไปที่หน้า Settings และดูว่าข้อมูล Company Profile และ Presets ยังคงอยู่เหมือนเดิม
- [ ] ไปที่ Health Check ใน Workspace ที่กู้คืนมา และรันตรวจสอบว่าสมบูรณ์หรือไม่

## 7. การทำงานแบบ File-First (External Editing)
- [ ] เปิดโฟลเดอร์ Workspace ด้วยโปรแกรมจัดการไฟล์ของระบบ (File Explorer / Finder)
- [ ] ทดลองเข้าไปแก้ไฟล์ Markdown หรือ YAML (.md / .yaml) เล็กน้อยด้วยโปรแกรม Text Editor อื่น
- [ ] กลับมาที่ ScopeFlow Thai และดูว่าแอปมีการอัปเดตข้อมูล (Refresh Index) ได้ถูกต้อง หรือรัน Health Check เพื่อดูการเปลี่ยนแปลง
